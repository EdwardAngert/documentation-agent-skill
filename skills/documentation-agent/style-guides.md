# Style Guide Selection and Enforcement

## Choose a Style Guide

### Common Technical Writing Style Guides

- **Google Developer Documentation Style Guide**: Modern, comprehensive, API-focused
- **Microsoft Writing Style Guide**: Software and tech products
- **Chicago Manual of Style**: General purpose, academic roots
- **AP Stylebook**: News/journalism style
- **Custom/Hybrid**: Combination with company-specific rules

### Selection Criteria

- Product type (API, UI, infrastructure)
- Audience (developers, end users, mixed)
- Industry standards
- Team size and enforcement capacity
- Existing content volume

## Style Guide Components

### Voice and Tone

- Audience-appropriate formality
- Active vs passive voice rules
- Second person vs third person
- Contractions policy
- Humor/personality guidelines

### Terminology

- Preferred terms list
- Prohibited terms
- Product-specific vocabulary
- Capitalization rules
- Abbreviation standards

### Formatting

- Heading structure (sentence case vs title case)
- Lists (when to use bullets vs numbers)
- Code formatting (inline vs blocks)
- UI element references
- File path conventions

### Mechanics

- Punctuation specifics
- Number treatment
- Date/time formats
- Link text guidelines
- Alt text requirements

## Enforce Your Style Guide

### Automated Checks

- Linters (Vale, write-good)
- CI/CD integration
- Pre-commit hooks
- Spell check
- Link validation

### Human Review

- Peer reviews
- Editorial passes
- Subject matter expert review
- User testing

### Documentation

- Style guide easily accessible
- Examples for each rule
- Decision rationale
- Update process
- Exception process

## Common Issues

### Inconsistent Terminology

- Problem: Same concept, different terms
- Solution: Terminology database, automated checks

### Style Drift

- Problem: New writers unaware of standards
- Solution: Onboarding documentation, templates

### Outdated Guidelines

- Problem: Style guide doesn't reflect current product
- Solution: Regular reviews, version control

### Over-prescription

- Problem: Style guide too detailed, hard to follow
- Solution: Focus on high-impact rules, provide rationale
